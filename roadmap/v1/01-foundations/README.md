# Foundations

## Objective

Build a strong understanding of the fundamentals that support every software engineering decision.

Rather than focusing on frameworks, this epic focuses on principles that remain valuable throughout an engineering career.

---

## Why?

Frameworks evolve quickly.

Fundamentals remain relevant for decades.

The goal of this epic is to understand how computers, networks, operating systems and algorithms work before building increasingly complex systems.

---

## Completion Criteria

By the end of this epic I should be able to:

- Explain Big-O notation and algorithmic trade-offs.
- Understand common data structures and when to use them.
- Explain how memory works.
- Understand processes and threads.
- Explain synchronization and concurrency basics.
- Understand networking fundamentals (HTTP, TCP, UDP, DNS).
- Understand how operating systems schedule work.
- Read and analyze performance bottlenecks.

---

## Learning Areas

- Algorithms
- Data Structures
- Networking
- Operating Systems
- Concurrency
- Performance Fundamentals

---

## Definition of Done

This epic is complete when I can confidently use these concepts to reason about engineering problems rather than simply remembering definitions.
